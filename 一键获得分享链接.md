# 一键获得分享链接（类似 Gemini 的链接）

不用自己部署服务器，用 **StackBlitz** 或 **CodeSandbox** 把项目放上去，就会自动得到一个**所有人能打开的链接**，并可以继续在线改代码、测试功能。

---

## 方法：用 StackBlitz 获得链接（推荐）

### 第一步：把项目放到 GitHub（若还没有）

1. 在 [github.com](https://github.com) 新建一个仓库（例如 `landscape-configurator`）。
2. 在本地项目文件夹打开终端，执行（把 `你的用户名` 和 `landscape-configurator` 换成你的）：

```bash
git init
git add .
git commit -m "init"
git branch -M main
git remote add origin https://github.com/你的用户名/landscape-configurator.git
git push -u origin main
```

若已有 GitHub 仓库，只需 `git push` 即可。

**注意**：`.env` 里有 Google 客户端 ID，通常不应提交。若仓库是私有的可以暂时提交；若是公开的，先不要提交 `.env`，后面在 StackBlitz 里手动填环境变量。

### 第二步：用 StackBlitz 打开并得到链接

1. 打开 **[stackblitz.com](https://stackblitz.com)**。
2. 点击 **“Import project”** 或 **“Clone from GitHub”**。
3. 填入你的仓库地址，例如：`https://github.com/你的用户名/landscape-configurator`。
4. 等 StackBlitz 拉取并安装依赖、启动项目（用 Vite 的话会自动跑起来）。
5. 在界面里找到 **“Open in New Window”** 或 **“Share”** / **“Preview”**，会得到一个**可分享的链接**，例如：
   - `https://landscape-configurator-xxx.stackblitz.io`
   - 或 `https://stackblitz.com/edit/landscape-configurator-xxx`

把这个链接发给别人，他们打开就能用，**不需要部署到 Vercel**，也不用你电脑开机。

### 第三步（可选）：让“上传到 Google Drive”在 StackBlitz 链接下也能用

1. 在 StackBlitz 项目里，打开 **左侧或底部设置/Environment/Env**，添加变量：
   - 名称：`VITE_GOOGLE_CLIENT_ID`
   - 值：你的 Google OAuth 客户端 ID（和本地 `.env` 里一样）
2. 在 [Google Cloud Console](https://console.cloud.google.com/) → 凭据 → 你的 OAuth 客户端 → **已授权的 JavaScript 源** 里添加：
   - `https://*.stackblitz.io`
   - 或 StackBlitz 给你的具体域名（例如 `https://landscape-configurator-xxx.stackblitz.io`）
3. 保存后，在 StackBlitz 里刷新预览页面，再试“上传到 Google Drive”。

## 和 Gemini 链接的对比

| 对比       | Gemini 分享链接           | 你的项目用 StackBlitz 链接      |
|------------|----------------------------|----------------------------------|
| 谁在运行   | Google 服务器              | StackBlitz 的服务器（浏览器里跑） |
| 链接形式   | `https://gemini.google.com/share/xxx` | `https://xxx.stackblitz.io` 或类似 |
| 所有人能开 | 是                         | 是                               |
| 你继续测试 | 在 Gemini 里改             | 在 StackBlitz 里改代码、保存、刷新预览 |

这样你就有**一个固定链接**可以发给别人用，同时还能在 StackBlitz 里继续改功能、测试，和“部署到公网”相比更接近“像 Gemini 那样直接给个链接”的体验。

---

## 备选：CodeSandbox

1. 打开 **[codesandbox.io](https://codesandbox.io)**。
2. **Import from GitHub**，填入你的仓库地址。
3. 等导入完成，CodeSandbox 会给出一个预览链接（例如 `https://xxx.codesandbox.io`），同样可以分享给任何人。

Google OAuth 配置方式同上：在 CodeSandbox 里加 `VITE_GOOGLE_CLIENT_ID`，并在 Google 控制台里把该域名加入“已授权的 JavaScript 源”。
